# AI 记忆沙盒

> 这不是一个 AI RPG。
> 它更像一个 **"世界容器（World Container）"**。
> AI 负责让世界活着，程序负责让世界不会遗忘。

---

> **AI 负责创造世界，Memory 负责延续世界，UI 负责让玩家看见世界。**
>
> 三者职责完全分离：
> - **AI**：创造与推演。
> - **Memory**：保存与回馈，不做解释。
> - **UI**：可视化世界，不做判断。

## 最高设计原则

> 如果程序未来需要新增一个功能，先问自己：
> **"这是在帮助AI记住，还是在替AI思考？"**
>
> ✅ 帮助 AI 记住——保存状态、展示状态、搜索历史、回填记忆 → **可以做**
> ❌ 替 AI 思考——计算好感、设计任务、决定剧情、判断事件、限制玩法 → **不要做**

## 核心理念

这个项目不是游戏引擎，不是RPG框架，也不是规则系统。

**它本质上是一个 AI 世界状态同步器（AI Memory Layer）。**

职责只有一句话：**AI负责创造世界，程序负责让AI永远记住自己创造过的世界。**

## 程序职责

1. 读取 AI 输出
2. 提取长期状态
3. 保存状态
4. 下轮反馈给 AI

**不做：** 剧情推演 / 规则判断 / 数值计算 / 玩法限制

## 哲学（重要）

## Prompt 首要指令

> 你是一个持续运行的世界模拟GM。不得控制玩家、不得强制剧情、NPC必须独立行动、世界必须保持一致性，所有长期状态必须通过STATE同步。

## GM 行为规则（Prompt 层）

这些规则写在 systemPrompt 中，不是代码逻辑。它们约束 AI 的行为方式，不限制内容自由。

1. **玩家控制权不可侵犯** — 不得替玩家做决定，只描述环境与后果
2. **不得强制剧情推进** — 不生成"必须完成/自动完成"的叙事结果
3. **NPC独立性原则** — NPC 有独立动机，不是玩家的服务型工具
4. **世界连续性原则** — 已确认的事实不得随意修改
5. **状态同步优先原则** — 所有影响长期世界的信息必须以 STATE 形式更新

### 程序永远不知道 Memory 里面保存的内容是什么意思

它只知道：

```
字段 → 保存 → 显示 → 反馈
```

不翻译字段名（AI 写 `cultivation` 就显示 `cultivation`，不翻译成"修为"）
不筛选字段（AI 创造多少字段就显示多少字段）
不预设字段（Prompt 里不列出需要 AI 输出哪些字段）

### 合并策略

AI 本轮没提到的字段 → 保留原值不动
AI 明确设为 `null` 或 `[]` → 更新
AI 提到的字段 → 覆盖
